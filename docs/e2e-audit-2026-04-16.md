## 1. Installation E2E (New Machine)

### Golden path (6 steps)

1. `uv tool install --from git+https://github.com/shawnoster/aya aya-ai-assist`
2. `aya init --label <name>`
3. `aya schedule install` (bundles: crontab + SessionStart/PreToolUse/PostToolUse hooks)
4. `aya pair --peer <name>` (optional, for relay)
5. Shell alias: `alias claude='claude --plugin-dir /path/to/aya'`
6. `aya status` (verify)

### Silent failures found

| Scenario | Location | Behavior | Severity | Status |
|----------|----------|----------|----------|--------|
| WSL without cron service | `install.py:257-267` | `crontab -l` fails silently; user gets misleading "Could not find 'aya' on PATH" | HIGH | ✅ Fixed in #228 |
| `gh` CLI missing | `scheduler/providers.py:41-56` | Returns `None` silently; GitHub PR checks disappear with no warning | MEDIUM | ✅ Fixed in #228 |
| `settings.json` missing | `install.py:380-381` | Returns `{}`; creates hooks anyway; recovers on save | LOW | accepted — low risk |
| coincurve build fails | `pyproject.toml:11` | Loud failure during `uv tool install`, but no pre-flight check or workaround docs | MEDIUM | ✅ Documented in #227 |

### Discoverability gaps

- `aya init` output has no next-steps guidance. ✅ Fixed in #227
- `/aya-setup` skill existed in `.claude/commands/` but was undiscoverable. ✅ Consolidated into `/aya` router skill (#231)
- Plugin alias (`--plugin-dir`) was undocumented in status output. ✅ README updated in #231

### WSL issues

✅ WSL cron `FileNotFoundError` now caught with correct error message and install guidance (#228).

## 2. Upgrade E2E (Existing Machine)

### Current path

1. `uv tool uninstall aya-ai-assist`
2. `uv tool install --force --from git+https://github.com/shawnoster/aya aya-ai-assist`
3. `aya status` (verify)
4. `aya schedule install` (re-install hooks — **not automatic**)

### What's safe

- **`~/.aya/` is fully preserved** across uninstall/reinstall. Profile, scheduler,
  alerts, activity, packets, config — all untouched.
- **Schema is forward-compatible.** Both profile.json and scheduler.json have
  `schema_version` fields with migration logic and graceful degradation warnings.
- **`aya schedule install` is idempotent.** Detects existing hooks via markers,
  returns `already_present=True` rather than failing.

### Gaps

| Issue | Severity | Detail | Status |
|-------|----------|--------|--------|
| No version staleness warning | MEDIUM | `aya version` only prints current version; no remote check | deferred |
| No post-upgrade hook re-install | HIGH | `aya-refresh` skill runs install but doesn't re-run `aya schedule install` | ✅ Fixed in #227 |
| Hook format not versioned | MEDIUM | `CANONICAL_HOOKS` has no version field; format changes won't auto-replace old hooks | deferred |
| Python 3.14 blocking | MEDIUM | coincurve 21.0.0 has no cp314 wheel; source build fails. Must pin to 3.12. Not documented for users. | ✅ Documented in #227 |

| Reply | `dispatch --in-reply-to` → `send --in-reply-to` | `aya_ack` | `/relay reply` | ✅ Renamed in #230 |

| Packet history | `packets` | `aya_packets` | — | ✅ MCP gap closed in #232 |

### Call patterns

- **Plugin skills** → MCP tools preferred, CLI as fallback. ✅ Updated in #237
